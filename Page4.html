📌 File Path System

    🔑 Big Truth

      A website is not just one file.
      It’s a collection of files:

    • HTML(Structure)
    • CSS(Styling)
    • JavaScript(Logic)
    • Images, Videos, Audios

      • 👉 These files live in folders, just like your laptop files.

    ❓ The Problem

      How does index.html know where to find logo.png or style.css?
      Answer: File Paths = exact address of files.

    🛤️ Relative File Paths (used 99% of the time)

      Relative path = start searching from the file you’re currently in.
      📂 Example Project:

      ✅ Scenario 1: Same Folder

      👉 Linking index.html to about.html

      ✅ Scenario 2: Going Down (Sub-Folder)

      👉 Using an image inside the images folder

      ✅ Scenario 3: Going Up (Parent Folder)

      👉 From pages/contact.html to images/logo.png

    🌍 Absolute File Paths

      These give the full address (with http:// or https://).
      Use only for external resources.

    🛡️ Why Relative Paths are Safer

      Browsers run in a sandbox (prison).
      They protect your computer from bad websites that might try to read your files.
      So → Always use relative paths for your files.

    📏 Rules of Thumb

    1. Your Files → Relative paths
      • filename.html (same folder)
      • folder/filename.html (down)
      • ../filename.html (up)
    2. Other Website’s Files → Absolute paths
      • https://example.com/file.png

    🖥️ Windows vs macOS/Linux File Paths

      Windows vs macOS/Linux File Paths
      Feature Windows macOS/Linux
      Root Drive Letter (C:) Forward Slash /
      Separator \ (backslash) / (forward slash)
      Example C:\Users\John\Documents\report.docx /Users/john/Documents/report.docx

    🏗️ HTML Boilerplate

      🧩 Why Needed?

        Browsers don’t guess → they need clear instructions.
        Boilerplate = starter template with all important info.

      🛠️ Explanation

      1. → Tells browser “This is HTML5, use standard rules.”
      2. → Language of page (important for SEO + accessibility).
      3. → Hidden info for browser.
      4. → Correctly shows all symbols (€, ₹, ©).
      5. → Mobile friendly, makes page responsive.
      6. < title > → Title shown in browser tab.
      7. < body > → Actual content (headings, images, text).

      🧪 But Websites Work Without Boilerplate?

        Yes, because browsers are forgiving. They add missing tags automatically.
        But… without boilerplate you risk:

      • ❌ Quirks Mode (unpredictable layouts)
      • ❌ Broken characters (€, é, ’)
      • ❌ Bad mobile experience
      • ❌ Poor SEO & accessibility

        • 👉 Boilerplate = strong foundation 🏠

    📄 Multi-Page Websites

      Websites are like books → multiple chapters (pages).
      Example: Home, About, Contact.

      Need two things:

      1. Consistent structure (same header & footer).
      2. Navigation links ( tags).
